\subsection{Definición de conceptos clave}
\begin{itemize}
    \item \textbf{Back-end:} Segmento lógico de una aplicación informática que se ejecuta en el servidor y procesa las reglas de negocio, la comunicación con la base de datos y la seguridad, permaneciendo invisible al usuario final.
    \item \textbf{Front-end:} Entorno visual y de presentación de la aplicación interactiva con la que el usuario final tiene contacto directo a través de navegadores web.
    \item \textbf{Base de Datos Relacional:} Sistema de organización de datos estructurados en filas y columnas que se interconectan mediante llaves primarias y foráneas asegurando la integridad referencial.
    \item \textbf{API REST:} Interfaz de programación de aplicaciones que utiliza los métodos del protocolo HTTP para intercambiar datos de manera ligera y estandarizada, usualmente en formato JSON.
\end{itemize}

\subsection{Explicación de tecnologías y herramientas utilizadas}
\begin{itemize}
    \item \textbf{Spring Boot:} Framework del ecosistema Java diseñado para simplificar la configuración y el despliegue de aplicaciones empresariales. Su enfoque de ``convención sobre configuración'' y servidores embebidos agiliza la creación de microservicios y APIs robustas.
    \item \textbf{Postgres:} Motor de base de datos de código abierto altamente confiable, caracterizado por su alto rendimiento, soporte multiusuario y estricto cumplimiento de las propiedades ACID.
    \item \textbf{JavaScript / Bootstrap:} Lenguaje de programación del lado del cliente acoplado con una librería de diseño responsivo que permite la creación de vistas adaptables a múltiples resoluciones de pantalla y dispositivos móviles.
\end{itemize}

\section{Normativas}
El diseño arquitectónico, la implementación lógica y el tratamiento de datos de la presente aplicación web de lectura métrica se alinean estrictamente con el marco jurídico nacional regulado por la Ley Federal del Derecho de Autor (LFDA) de México. Debido a las características particulares del sistema ---el cual opera como una biblioteca centralizada de indicadores sin almacenamiento ni distribución de archivos de obras protegidas---, se identifican y aplican las siguientes disposiciones normativas específicas:

\begin{itemize}
    \item \textbf{Protección del Contenido Generado por el Usuario (Módulo de Notas y Acotaciones):} Con base en el \textbf{Artículo 14, Fracción VIII}, el sistema reconoce que las anotaciones, interpretaciones, comentarios y revisiones literarias que los usuarios registren de forma individual en la plataforma constituyen obras protegidas, ya que entrañan la creación de un trabajo original diferenciado. Asimismo, la recopilación de fragmentos textuales para la crítica e investigación científica o literaria queda amparada bajo el \textbf{Artículo 148, Fracción III}, obligando al sistema a proveer los campos necesarios para citar invariablemente la fuente original.
    
    \item \textbf{Exclusión de Protección en Mecánicas lógicas (Módulo de Gamificación):} Las reglas del sistema, fórmulas de puntaje, algoritmos de progresión y los esquemas lógicos diseñados para la gamificación de la lectura métrica no constituyen objeto de protección por derecho de autor según lo estipulado en el \textbf{Artículo 14, Fracciones I y III}. Por ende, la protección legal de este módulo se restringe de forma exclusiva a la codificación imperativa del software (\textit{back-end} y \textit{front-end}) y no a las ideas metodológicas subyacentes.
    
    \item \textbf{Libre Indexación de Metadatos (Buzón de Recomendaciones y Comunidad):} El catálogo comunitario de recomendaciones opera de manera legítima bajo el \textbf{Artículo 14, Fracción V}, el cual determina que los nombres y títulos aislados de las obras no son objeto de protección autoral. Esto faculta a la base de datos relacional para almacenar, listar y desplegar libremente los títulos de los libros recomendados por los usuarios dentro de la comunidad asíncrona.
    
    \item \textbf{Interoperabilidad y Pruebas de Seguridad en la Infraestructura:} La arquitectura lógica del sistema observa las flexibilidades de los \textbf{Artículos 114 Quáter (Fracciones I y III)} y \textbf{114 Quinquies}. Estas normas facultan la realización de procesos de ingeniería inversa de buena fe destinados exclusivamente a lograr la interoperabilidad de la aplicación web con otros programas de software de lectura externa, así como la ejecución de pruebas y auditorías destinadas a corregir la seguridad de la base de datos centralizada.
    
    \item \textbf{Ausencia de Actos de Reproducción o Distribución Patrimonial:} En estricto apego al \textbf{Artículo 16 (Fracciones II, V y VI)}, la plataforma se exime de la solicitud de licencias de explotación de obras o del pago de remuneraciones compensatorias descritas en el \textbf{Artículo 40}, en virtud de que la infraestructura web centralizada no realiza copias, reproducciones temporales, almacenamiento provisional, ni distribución al público de los contenidos íntegros de los libros, limitándose únicamente al procesamiento de métricas cuantitativas de lectura.
\end{itemize}
